# Scripts and Bindings

mpv loads scripts only from a small set of well-known locations. This page
explains the loading model used in this NixOS configuration, the scripts that ship enabled,
the inline Lua hooks, and the keybindings written to `~/.config/mpv/input.conf`.

## Script Loading Model

mpv only auto-loads scripts from `~/.config/mpv/scripts/`. It does not scan
the system profile, so this NixOS configuration exposes two independent knobs that look
interchangeable but are not:

- `programs.mpv.extended.extraScripts` (NixOS): Appends packages to `environment.systemPackages`. The script files end up under the system profile but mpv does **not** load them.
- `programs.mpv.extended.extraPackages` (NixOS): Same mechanism, named differently to signal that the entries are auxiliary tools rather than scripts.
- `programs.mpv.scripts` (HM): Appends script packages to mpv's per-user script search path. mpv loads these at startup.
- `xdg.configFile."mpv/scripts/<name>.lua"` (HM): Drops a raw Lua file into `~/.config/mpv/scripts/`. mpv loads it at startup.

To make a script actually run, place it in the HM layer (`programs.mpv.scripts`
or `xdg.configFile`). The NixOS layer's `extraScripts` is useful only when the
goal is to make the package available to other tools or to reference it from
custom mpv invocations. See [customizing.md](5. customizing.md) for the recommended
patterns.

## MPV Bundled Scripts

The HM module sets `programs.mpv.scripts` to load these packages:

- `mpvScripts.mpris`: Exports the standard MPRIS DBus interface. Allows desktop environments and `playerctl` to control playback and read metadata.
- `mpvScripts.reload`: Watches stalled streams and re-issues the load when buffering hangs, instead of leaving mpv frozen on a dead segment.

Two further packages are listed in the same `scripts` block but commented out
pending follow-up evaluation: `mpvScripts.autoload` (auto-queue the next file
in the directory) and `mpvScripts.mpv-cheatsheet` (overlay cheat sheet of
bindings). They do not load until the comment is removed.

To enable or disable any of them or adding more, see [customizing.md](5. customizing.md).

## Inline Lua Hooks

The HM module writes two Lua scripts via `xdg.configFile` to
`~/.config/mpv/scripts/`. Both load alongside the bundled `mpvScripts.*`
packages.

### `block-images.lua`

The script:

- Registers an `on_preloaded` hook.
- Inspects each playlist entry's path extension.
- Removes the entry from the playlist before mpv begins decoding when the
  extension matches a blocked image type, and logs a warning to mpv's message
  log.

mpv otherwise treats image files as single-frame video, which is rarely the
desired behavior in a mixed-content playlist (for example a directory of
mostly videos with a few thumbnails). The hook drops them silently rather
than letting mpv pause on a still frame.

### `ytdlp-cookies.lua`

The script resolves a browser at mpv startup and appends
`cookies-from-browser=<browser>` to `ytdl-raw-options`, so streaming URLs
that need session cookies (members-only videos, region-locked content, etc.)
authenticate with the active browser profile. Resolution order:

- `MPV_YTDLP_BROWSER` (dedicated override): Set to a browser name to force
  the value, or to the empty string to disable cookie passthrough entirely.
- `$BROWSER` (set by the XDG defaults machinery): Used as a fallback when
  `MPV_YTDLP_BROWSER` is unset. Picks up whatever the host configured as the
  default web browser.
- `--ytdl-raw-options=cookies-from-browser=<browser>` on the mpv command
  line: Detected by the script and treated as authoritative; the script
  suppresses its own append in this case.

If neither env var is set and no CLI flag is passed, the script logs at
verbose level and leaves `ytdl-raw-options` untouched, so mpv runs without
cookies rather than failing.

## Keybindings

The HM module writes the following entries to `~/.config/mpv/input.conf` via
`programs.mpv.bindings`.

- `q`: `quit`
- `n`: `playlist-next`
- `p`: `playlist-prev`
- `m`: `cycle mute`
- `v`: `cycle video`
- `a`: `cycle audio`
- `j`: `seek 5`
- `h`: `seek 5`
- `k`: `seek -5`
- `l`: `seek -5`
- `[`: `add speed -0.1`
- `]`: `add speed 0.1`

### Notes on the Mapping

- `j`/`k` intentionally duplicate `h`/`l`: all four vim-direction keys seek ±5 s.
  Volume is handled via the scroll wheel and does not need a keyboard binding.
- `h`/`l` are inverted relative to standard vim direction (`h` seeks forward,
  `l` seeks backward). `j`/`k` follow the same convention (`j` forward, `k` backward).
- `[`/`]` follow mpv upstream defaults: `[` slows playback, `]` speeds it up.
- mpv's non-overridden defaults remain accessible.
