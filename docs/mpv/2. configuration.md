# Configuration

When a host has `programs.mpv.extended.enable = true`, the Home Manager app
module activates and writes a managed `~/.config/mpv/mpv.conf`. This page
documents every key set by the module, what it does, and the rationale for
values that diverge from upstream defaults. Cross-link the canonical mpv manual
at <https://mpv.io/manual/stable/> for option semantics.

## Files Produced

- `~/.config/mpv/mpv.conf`: `programs.mpv.config` in the HM module
- `~/.config/mpv/input.conf`: `programs.mpv.bindings` in the HM module (see [scripts-and-bindings.md](3. scripts-and-bindings.md))
- `~/.config/mpv/scripts/playlist_filter.lua`: `xdg.configFile."mpv/scripts/playlist_filter.lua"` in the HM module (see [scripts-and-bindings.md](3. scripts-and-bindings.md))
- `~/.config/mpv/scripts/ytdlp_cookies.lua`: `xdg.configFile."mpv/scripts/ytdlp_cookies.lua"` in the HM module (see [scripts-and-bindings.md](3. scripts-and-bindings.md))
- Loaded scripts under `~/.config/mpv/scripts/`: `programs.mpv.scripts` (see [scripts-and-bindings.md](3. scripts-and-bindings.md))

## Settings Reference

### Playback and Window

- `osc = no`: Disables the built-in OSC. modernz provides the replacement and sets this at runtime; the config value makes the intent explicit.
- `pause = no`: Start playing immediately when a file is opened.
- `save-position-on-quit = no`: Do not auto-resume on next launch. Fresh playback is preferred over resume state.
- `watch-later-options-remove = sub-pos`: Drops `sub-pos` from the persisted watch-later option list. modernz's `sub_margins` feature drives subtitle position dynamically; leaving `sub-pos` in the list causes modernz to emit a startup warning (`sub_margins conflict: add watch-later-options-remove=sub-pos to mpv.conf`).
- `volume = 60`: Default starting volume. Reasonable headroom over typical desktop levels.
- `border = no`: Frameless window. modernz renders its own controls.
- `keepaspect-window = no`: Allow the window to be resized freely. Aspect ratio is preserved inside the frame.
- `cache = yes`: Enable mpv's input cache. Helpful on streaming sources and slow filesystems.

### Quality

- `profile = high-quality`: mpv's built-in quality profile. Enables better scalers, dithering, and color management. See the upstream profile listing.

### GPU and Hardware Decoding

- `vo = gpu-next`: The next-generation unified video output. Required for `profile=high-quality`. Supersedes `vo=gpu`.
- `gpu-context = auto`: Let mpv pick a supported windowing context (X11 or Wayland) at runtime.
- `hwdec = auto`: Negotiate hardware decode at runtime. Falls back to software when no path is available.
- `hwdec-codecs`: Not set; mpv's upstream default applies (`h264,vc1,hevc,vp8,vp9,av1,prores,prores_raw,ffv1,dpx`). Earlier revisions extended the list with `mpeg2video,mpeg4,msmpeg4v2,msmpeg4v3`, but `mpeg4` Simple Profile (the dominant variant in the wild) is rejected by every hwaccel backend on this hardware and only produced probe noise without a payoff.

### yt-dlp

- `ytdl = yes`: Enable mpv's youtube-dl/yt-dlp passthrough. Required for streaming URLs.
- `ytdl-format = best`: Single-stream selection. Trades muxing complexity for predictability.
- `ytdl-raw-options` (dynamic): Not set in `mpv.conf`. The `ytdlp_cookies.lua` script appends `cookies-from-browser` at startup by resolving the browser from env vars in order: `MPV_YTDLP_BROWSER` (dedicated override), then `$BROWSER` (set by the XDG defaults machinery). Set `MPV_YTDLP_BROWSER=""` to disable cookie passthrough. Passing `--ytdl-raw-options=cookies-from-browser=X` at the mpv command line is also respected and suppresses the script.

### Profiles

The HM module declares two named profiles via `programs.mpv.profiles`. They are
inactive by default and activate with `--profile=<name>` (or the `-p` short
form). For runtime mode cycling and the underlying `playlist_filter.lua`, see
[scripts-and-bindings.md](3. scripts-and-bindings.md).

- `vid` (skip image files in playlists)
  - `script-opts = playlist_filter-mode=block-images`: Sets the `playlist_filter` script's startup mode to drop image entries from the playlist, leaving videos and audio.
- `img` (image viewer: skip non-image files, hold each image until manual advance, loop playlist)
  - `script-opts = playlist_filter-mode=images-only`: Inverse filter; drops non-image entries.
  - `image-display-duration = inf`: Hold each image until manual advance (`n`/`p`).
  - `loop-playlist = inf`: Cycle back to the first entry after the last image.
  - `osd-level = 0`: Suppress the on-screen display so the image fills the window cleanly.

Both profiles activate `script-opts` rather than `script-opts-append`, so the
full `script-opts` value is replaced when the profile is active. There is only
one consumer (`playlist_filter`) today, so this is currently invisible; a
future second consumer would require switching to the append form.

The HM module also installs `mpv-vid` and `mpv-img` shell wrappers as a
shorthand for the profile flags. See
[scripts-and-bindings.md](3. scripts-and-bindings.md) for the full list of
profile invocation paths.
