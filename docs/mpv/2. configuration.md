# Configuration

When a host has `programs.mpv.extended.enable = true`, the Home Manager app
module activates and writes a managed `~/.config/mpv/mpv.conf`. This page
documents every key set by the module, what it does, and the rationale for
values that diverge from upstream defaults. Cross-link the canonical mpv manual
at <https://mpv.io/manual/stable/> for option semantics.

## Files Produced

- `~/.config/mpv/mpv.conf`: `programs.mpv.config` in the HM module
- `~/.config/mpv/input.conf`: `programs.mpv.bindings` in the HM module (see [scripts-and-bindings.md](3. scripts-and-bindings.md))
- `~/.config/mpv/scripts/block-images.lua`: `xdg.configFile."mpv/scripts/block-images.lua"` in the HM module
- `~/.config/mpv/scripts/ytdlp-cookies.lua`: `xdg.configFile."mpv/scripts/ytdlp-cookies.lua"` in the HM module (see [scripts-and-bindings.md](3. scripts-and-bindings.md))
- `~/.config/mpv/hwdec-codecs.conf`: `xdg.configFile."mpv/hwdec-codecs.conf"` in the HM module, included from `mpv.conf` via `programs.mpv.includes`
- Loaded scripts under `~/.config/mpv/scripts/`: `programs.mpv.scripts` (see [scripts-and-bindings.md](3. scripts-and-bindings.md))

## Settings Reference

### Playback and Window

- `osc = yes`: Show the on-screen controller. mpv default; set explicitly so the value is documented.
- `pause = no`: Start playing immediately when a file is opened.
- `save-position-on-quit = no`: Do not auto-resume on next launch. Fresh playback is preferred over resume state.
- `volume = 60`: Default starting volume. Reasonable headroom over typical desktop levels.
- `border = no`: Frameless window. The mpv OSC carries the controls.
- `keepaspect-window = no`: Allow the window to be resized freely. Aspect ratio is preserved inside the frame.
- `cache = yes`: Enable mpv's input cache. Helpful on streaming sources and slow filesystems.

### Quality and Frame Timing

- `profile = high-quality`: mpv's built-in quality profile. Enables better scalers, dithering, and color management. See the upstream profile listing.
- `video-sync = display-resample`: Resample audio to the display refresh rate. Removes judder on high-refresh displays.
- `interpolation = yes`: Frame-blend between source frames at the resampled rate. Pairs with `tscale`.
- `tscale = oversample`: Temporal scaler used when `interpolation=yes`. Sharp edges, minimal smear.

### GPU and Hardware Decoding

- `vo = gpu-next`: The next-generation unified video output. Required for `profile=high-quality`. Supersedes `vo=gpu`.
- `gpu-context = auto`: Let mpv pick a supported windowing context (X11 or Wayland) at runtime.
- `hwdec = auto`: Negotiate hardware decode at runtime. Falls back to software when no path is available.
- `hwdec-codecs`: Not set in `mpv.conf`. Managed via `hwdec-codecs-append` lines in `~/.config/mpv/hwdec-codecs.conf` (included at the end of `mpv.conf`), which extends the upstream default list without replacing it.

### yt-dlp

- `ytdl = yes`: Enable mpv's youtube-dl/yt-dlp passthrough. Required for streaming URLs.
- `ytdl-format = best`: Single-stream selection. Trades muxing complexity for predictability.
- `ytdl-raw-options` (dynamic): Not set in `mpv.conf`. The `ytdlp-cookies.lua` script appends `cookies-from-browser` at startup by resolving the browser from env vars in order: `MPV_YTDLP_BROWSER` (dedicated override), then `$BROWSER` (set by the XDG defaults machinery). Set `MPV_YTDLP_BROWSER=""` to disable cookie passthrough. Passing `--ytdl-raw-options=cookies-from-browser=X` at the mpv command line is also respected and suppresses the script.

## Rationale Notes

- `interpolation=yes` plus `video-sync=display-resample` plus
  `tscale=oversample` is a coherent set; changing one without the others
  produces different artifacts. Keep them aligned.
